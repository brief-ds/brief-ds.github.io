## The four experiments

Andrej Karpathy did a self-contained 200-line [`microgpt.py`](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95) in Python that trains and makes inference from the GPT-2 model, the second generation of the model behind ChatGPT.

The Karpathy's version only works with scalar values, updating every model coefficient one by one. We changed the program a few times and timed it. From setup to setup, the acceleration is generally in the ballpark of 20 times if we use the vector extension on the hardware architecture, and a further 5 times if we change the model to a simpler one.

|  version  |  description  | number of lines (the fewer the simpler) | run time (the lower the better) | optimised loss (the lower the better) |
| --------- | -------------- | ----------- | ---------- | ---------- |
| scalar    |  the original version, updating model coefficient one by one   |      |  160s       |   2.66     |
| vector    |  uses vector extension on the hardware architecture, updating whole array of model coefficients at once | 164      |   7s        |   2.63     |
| rnn       |  changes the model to vanilla recurrent network, and uses vector extension |   118    |  1.2s    |   2.35     |
| rnn_att   |  as the `rnn` version but at each step, sparsely select and update coordinates of the state vector |  119     |  1.2s       |   2.44     |

The [code](https://github.com/jli05/microgpt) is in the `scalar`, `vector`, `rnn`, `rnn_att` branches respectively for the experiments. One will have to clone the vector-capable [micrograd repository](https://github.com/brief-ds/micrograd) and check out the `att` branch to work with it.

```sh
mkdir src; cd src
git clone https://github.com/jli05/microgpt
git clone https://github.com/brief-ds/micrograd
cd microgpt
python3 -m venv venv
. venv/bin/activate
cd ../micrograd && git checkout att && pip3 install . && cd ../microgpt

git checkout scalar
time python3 microgpt.py

git checkout vector
time python3 microgpt.py

# ...
```

## Vector extensions

Various architecture support a vector extension for vector operations, for example [AVX](https://en.wikipedia.org/wiki/Advanced_Vector_Extensions) for x86_64, [SVE](/2025/10/15/sve.html) for ARM, and the [V extension](/2025/10/31/rv64-v.html) for RISC-V.

The micrograd machine learning library would delegate _all_ numerical compute on arrays to NumPy. Simple functions on NumPy `array` would leverage the vector extension on hardware. Matrix-vector multiplication, matrix-matrix multiplication and alikes would be delegated to the [Basic Linear Algebra Subprograms (BLAS)](https://en.wikipedia.org/wiki/Basic_Linear_Algebra_Subprograms) further by NumPy.

The Basic Linear Algebra Subprograms (BLAS) are compiled from C and assembly into machine code, which usually make full use of any vector extension on hardware.

We compiled a version of microgpt written in C, comparable to the original "scalar" version that carries out update of coefficients one by one. The run time was 24s. This shows that as long as hardware acceleration and a C numerical library are leveraged under the hood, it is OK to write a compute-heavy program in Python.
