## Overview
Andrej Karpathy did a self-contained 200-line [`microgpt.py`](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95) that trains and makes inference from the GPT-2 model, the second generation of model behind ChatGPT.

The Karpathy's version only works with scalar values, single numbers one by one in the artificial neural network. We will first modify it to a vector version, to realise about 20 times acceleration, to the full extent that the [Advanced Vector Extensions (AVX)](https://en.wikipedia.org/wiki/Advanced_Vector_Extensions) from Intel and Advanced Micro Devices (AMD) was designed for. Then we change the architecture of the artificial neural network to simpler forms, for another 6 times acceleration.

Our [microgpt code](https://github.com/jli05/microgpt) is in the `scalar`, `vector`, `rnn`, `rnn_att` branches respectively for the experiments. One'll have to clone the vector-capable [micrograd repository](https://github.com/brief-ds/micrograd) and check out the `att` branch to work with it.

The optimised loss is the lower the better. But given the extremely small model size and data set size, as long as they are in a ballpark, the models are doing comparable.

|  version  |   no of lines  | run time (the lower the better) | optimised loss (the lower the better) |
| --------- | -------------- | ----------- | ---------- |
| scalar    |                |  160s       |   2.66     |
| vector    |   164          |   7s        |   2.63     |
| rnn       |   118          |  1.2s       |   2.35     |
| rnn_att   |   119          |  1.2s       |   2.44     |
